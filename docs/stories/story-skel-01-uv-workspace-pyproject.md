## Acceptance criteria (BDD — machine-verifiable)

```
Given the repo root has the new pyproject.toml with [tool.uv.workspace] declared
When  `uv sync` runs at the repo root
Then  exit code is 0
And   `.venv/` is created at repo root

Given uv sync has completed
When  `uv run python -c "import splunkgate_core, splunkgate_judges, splunkgate_mw, splunkgate_mcp, splunkgate_eval; print('ok')"` runs
Then  exit code is 0
And   stdout contains the literal string "ok"

Given the workspace is synced
When  `uv run pytest packages/splunkgate_core/tests/test_skeleton.py -q` runs
Then  exit code is 0
And   stdout contains "1 passed"

Given the .python-version file is read
When  `cat .python-version | tr -d '[:space:]'` runs
Then  stdout is exactly "3.13"

Given the LICENSE file is at repo root
When  `head -3 LICENSE | grep -ciE '(apache license)'` runs
Then  stdout is "1"
(Note: standard Apache-2.0 LICENSE template from apache.org starts with two blank lines before the "Apache License" header — check the first 3 lines, not just line 1.)

Given the lockfile is committed
When  `test -f uv.lock` runs
Then  exit code is 0

Given the .gitignore is committed
When  `grep -cE '^(\.venv|__pycache__|\.pytest_cache|\.mypy_cache|\.ruff_cache)' .gitignore` runs
Then  stdout is "5"

Given the workspace members are declared
When  `uv run python -c "import tomllib; d=tomllib.load(open('pyproject.toml','rb')); print(len(d['tool']['uv']['workspace']['members']))"` runs
Then  stdout contains "5"
```

Every criterion must be checkable by running a command. Prose-only criteria = blocked.


## Notes for coding agent

- Per `docs/architecture.md` § "Stack (locked)", language is Python 3.13+ because `splunk-sdk-python 3.0.0` hard-requires it. Do not weaken this constraint.
- Per `docs/architecture.md` § "Stack (locked)", package manager is `uv` (ADR-001). Do not introduce `poetry`, `pdm`, `hatch`, or `setuptools`-only configs.
- Per `docs/architecture.md` § "Repo structure (monorepo)", the workspace has exactly 5 members: `packages/splunkgate_core/`, `packages/splunkgate_judges/`, `packages/splunkgate_mw/`, `packages/splunkgate_mcp/`, `eval/`. `splunk_apps/splunkgate_app/` is intentionally NOT a uv workspace member — it is a Splunk app, not a Python package.
- Per `docs/architecture.md` § "Stack (locked)" ADR-002, this monorepo lives in a single repo so cross-surface PRs land atomically.
- Use `[tool.uv.sources]` with `<dep> = { workspace = true }` for intra-workspace deps (see uv workspace docs via `mcp__context7__resolve-library-id` then `mcp__context7__query-docs` for `uv` if unfamiliar).
- Use `src/` layout consistently — pyproject.toml needs `[tool.hatch.build.targets.wheel]` `packages = ["src/<pkg>"]` (or equivalent setuptools `package-dir` mapping) so editable installs resolve the right import path.
- Do NOT add runtime deps in this story — that lands in EPIC-03/04 stories. Each pyproject is essentially `[project]` + `name` + `version = "0.0.1"` + `requires-python = ">=3.13"` + empty `dependencies = []`.
- Do NOT add dev deps (pytest, ruff, mypy) in this story — story-skel-02 handles ruff/mypy and story-cicd-02 already brought pytest in via the build pipeline. If `pytest` is not yet on the workspace, add it as a top-level dev dep ONLY in the root `pyproject.toml` under `[dependency-groups]` `dev` so the skeleton test can run.
- LICENSE must already be Apache-2.0 (confirmed present per `docs/architecture.md` checklist line "Public GitHub repo with Apache-2.0 license"). If the file is missing, restore from the standard Apache-2.0 text — do not invent a license header.
