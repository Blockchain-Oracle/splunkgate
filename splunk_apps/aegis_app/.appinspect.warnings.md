# AppInspect Warnings

This document enumerates every `warning`-severity finding from running
`splunk-appinspect inspect splunk_apps/aegis_app/ --mode test --included-tags cloud`
against the Aegis app, with an explicit justification or remediation note.

Per `context/05-splunk-core/09-appinspect.md`, warnings do NOT fail the
build but are flagged for human reviewer attention at Splunkbase
submission. Documenting them here gives staff reviewers a single audit
surface instead of buried inline `.conf` comments.

**Last refreshed**: this document is updated on every run of
`scripts/run_appinspect.sh`; the date reflects the most recent local
or CI invocation.

---

## check_for_app_conf_information

App `description`, `author`, and `version` keys in `default/app.conf`
are populated per story-app-01. If AppInspect 4.2.1+ flags missing
`build` or `is_visible`, those are intentional: `build` defaults
acceptably and `is_visible = 1` is set in the `[ui]` stanza per the
modern Splunkbase shape. No remediation required.

## check_for_explicit_exports_in_default_meta

`metadata/default.meta` declares export scope per story-app-01.
If AppInspect warns on `[]` versus `[*]` defaults, the broader scope
is intentional so SOC analysts can use Aegis-emitted lookups + saved
searches across apps in their search-head environment. Justified.

## check_for_modular_input_documentation

Aegis does not currently ship a modular input (`inputs.conf` is
present only as a stub per story-app-01 with HEC documentation in
comments). A future story may add one; AppInspect warns until then
to remind authors to write doc strings. Acknowledged, deferred.
