# AppInspect Warnings

This document is the canonical audit surface for every `warning`-severity
finding from a `splunk-appinspect inspect splunk_apps/splunkgate_app/` run.
Each warning gets a `## <check_name>` section with a 2-3 sentence
justification or remediation note so a Splunkbase reviewer doesn't have
to chase buried inline `.conf` comments.

**Population:** this document is empty until the first AppInspect run
against the deployed app. The runner script
(`scripts/run_appinspect.sh`) currently does NOT auto-write to this
file; populating it is a maintainer task each time a real
`splunk-appinspect inspect ...` produces new warnings. See
`docs/stories/story-app-11-...md` and `docs/cicd-spec.md` for the
hand-off contract.

**When warnings appear:** append a `## <real_check_name>` section.
Examples of real AppInspect check names (verbatim from
`context/05-splunk-core/09-appinspect.md`): `check_app_version_number`,
`check_for_supported_tls`, `check_for_modular_inputs_logging`. Use
the exact name `splunk-appinspect` reports — do not paraphrase, and do
not invent plausible-sounding names.
